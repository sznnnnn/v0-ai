# Google Design Sprint (Jake Knapp)

Compress months of work into five days: understand the problem, sketch solutions, decide on one, build a prototype, and test with real users. Use when facing a big product decision and you need fast, validated direction.

## Steps

1. **Monday — Map** — define the challenge, pick a target user and moment, set a sprint goal.
2. **Tuesday — Sketch** — each team member sketches detailed solution ideas individually.
3. **Wednesday — Decide** — vote on the best ideas, pick a winner, create a storyboard for the prototype.
4. **Thursday — Prototype** — build a realistic, clickable prototype (fake it, don't build it).
5. **Friday — Test** — run 1-on-1 usability sessions with 5 target users; capture patterns and insights.

## Output Format

A sprint summary with: the sprint goal, the winning solution storyboard, the prototype link, user test findings organized by pattern, and a clear go/pivot/kill recommendation.
